Changelog depuis le dernier SVN

Note for ffmpeg users :
    With ffmpeg, video deinterlacing seems automatic.
    So the deinterlacing options are totally disabled, even if they are still present in dialogs

French version / Correction de bug :
    -Corrige le bug lié à l'anamorphisme des videos 16/9 stockée en 640x480 (SESTAY:852x480 au lieu de 853x480)
    -Corrige des problemes de déplacement dans les fichiers vidéos (seek)
    -Désactive la zone language quand l'inclusion du son est déselectionné dans la boite de génération de vidéo
    -Corrige l'affichage des informations quand on décoche l'inclusion du son dans la boite de génération de vidéo
    -Corrige plusieurs plantages aléatoires sous Windows liés au multi-thread lors du parcours de répertoire avec le browser
    -A cause d'un trop grand nombre de problèmes avec libav 9.x sous windows, bascule les dll du paquets Windows sur celles de ffmpeg/libav 9 (Zeranoe)
    -Corrige le bug de repositionnement des blocs à la position par défaut lors de la sélection d'un style dans la boite slide
    -Corrige le bug d'inactivité du raccourcie suppr dans la boite slide quand la sélection porte sur un plan

English version / Bug fixes :
    -Fixed bug related to anamorphism 16/9 videos stored 640x480 (SESTAY: 852x480 instead of 853x480)
    -Fixes problems in moving video files (seek)
    -Disable language area when the sound inclusion is deselected in the render dialog
    -Corrects the display of information when uncheck the inclusion of sound in the render dialog
    -Fixes several random crashes on Windows related to multi-thread when browsing directories with the browser
    -Because of too many problems with libav 9.x on Windows, switch dll used on Windows packages to ffmpeg/libav 9(Zeranoe) version
    -Fixed bug of positioning blocks to default position when you select a style in the slide dialog
    -Fixed bug of inactif del shortcut in slide dialog when the selection is a shot

French version / Nouveautés :
    -Pour limiter les problèmes, retire le codec audio AAC des AVI et le codec video divx/xvid des MKV
    -Bascule tous les formats restant vers le nouveau moteur d'encodage (supprime l'ancien moteur)
        => Pour libav 0.8.x, tous semble ok
        => Pour libav 9.x, certains problèmes subsistent :
            Error reading AAC channels in all file (read as mono)
            VLC+MKV error with h264 PQ and HQ preset
            webm error
            theora error/warning on reading (on first keyframe + read timestamp)
        => Pour ffmpeg/libav 9.x, certains problèmes subsistent :
            Error reading AAC channels in adts file
    -Nettoie le code pour ne plus embarquer de preset ni chercher des encodeurs externes
    -Retire l'option concernant l'utilisation des threads pour les opérations de pipe de la boite de configuration (ancien encoder)
    -Retire l'option de positionnement en haut dans le cas d'export en mode HTML5 (trop compliqué pour pas grand chose)
    -Change l'extension des fichiers adts en .aac (plus conforme au standard)

English version / New features :
    -To minimize problems, removes AAC audio codec in AVI and divx/xvid video codec in MKV
    -Switches all remaining formats to the new encoding engine (remove the old engine)
        => For libav 0.8.x, all seems ok
        => For libav 9.x, some problems remain:
            Error reading AAC channels in all file (read as mono)
            VLC+MKV error with h264 PQ and HQ preset
            webm error
            theora error/warning on reading (on first keyframe + read timestamp)
        => For ffmpeg/libav 9.x, all seems ok
            Error reading AAC channels in adts file
     -Cleans the code to no longer include preset and external encoders
     -Removes the option for the use of threads for pipe operations in the configuration dialog (old encoding)
     -Removes the option of positioning up in the case of export in HTML5 mode (too complicated for not much)
     -Change the file extension of ADTS file to .aac (more standard compliant)
