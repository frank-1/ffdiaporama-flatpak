[Commit: 08/11/2013]
French version
    Correction de bug :
        -Ne pose plus la question de fois, lors de l'ouverture d'un projet, si celui n'existe plus et que l'utilisateur répond non à la proposition de sélectionner un autre fichier
        -Corrige une fuite de mémoire qui survenait en preview et en génération avec des images statiques
    Nouveautés :
        -Accélère la réouverture des projets contenant des vidéos (ne teste plus les vidéos lors des ouvertures de projet)
        -Retire les thumbnails et les propriétés étendues des objets MediaFiles de la mémoire
        -Retire la reduction de qualité des thumbnails pour les systèmes 32 bits quand le nombre d'objets dans un répertoire est important (n'est plus nécessaire)
English version
    Bug fixes:
        -Do not ask the question once more, at the opening of a project if it no longer exists and that the user answers no to the proposal to select another file
        -Fixed a memory leak that occurred in preview and render with static images
    New features:
        -Accelerates reopening projects with videos (no longer tested videos when reopening project)
        -Removes thumbnails and extended properties of MediaFiles objects from memory
        -Removes the reduction of quality of thumbnails for 32-bit systems when the number of objects in a directory is important (no longer needed)

[Commit: 04/11/2013]
    French version
        Correction de bug :
            -Corrige l'erreur lors des générations en webm, ogv et ogg avec libav 0.8 : remplace AV_SAMPLE_FMT_FLT par AV_SAMPLE_FMT_S16 pour le codec OGG
        Nouveautés :
            - Réorganise les données à l'intérieur des objets afin de pouvoir a terme les supprimer de la mémoire et ne les avoir plus que dans la base de données
    English version
        Bug fixes:
            -Fixed the error when rendering to webm, ogv and ogg with libav 0.8: replaces AV_SAMPLE_FMT_FLT by AV_SAMPLE_FMT_S16 for OGG codec
        New features:
            - Arranges the data inside objects in order soon to remove them from memory and only have them in the database

    French version
        Correction de bug :
            -Corrige le plantage lors de l'annulation de l'insertion d'un objet dans la boite d'edition des slides [Bug chrisbra 01/11/2013]

            -Change all the source code to better take into account the differences between libav and ffmpeg
        New features:
            -Add a files table and remove filepath information from all objects derivated from cBaseMediaFile
